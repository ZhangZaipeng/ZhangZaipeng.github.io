搭建ngrok服务，实现内网穿透

自己的服务器到期，转移自己博客内容至此。

域名为：liuanhuaming.top。此篇博文以liuanhuaming.top为准。

       ngrok是一个反向代理，它能够让你本地内网的web服务或tcp服务通过公共的端口

和外部建立一个安全的通道，使得外网可以访问本地的计算机服务。

服务器环境：Centos 7
客户端环境：Windows 10

软件版本：
[root@xifeng ~]# git –version
git version 1.8.3.1
[root@xifeng ~]# go version
go version go1.8.3 linux/amd64

一、添加域名解析

        添加两个A记录，把域名解析到自己服务器上，用来关联ngrok服务。
比如我的域名添加：ngrok.liuanhuaming.top和*.ngrok.liuanhuaming.top

二、安装go环境

[root@xifeng ~]# yum install golang -y      #安装golang
[root@xifeng ~]# go version                 #查看版本
go version go1.8.3 linux/amd64    
[root@xifeng ~]# vi .bash_profile           #编辑.bash_profile，添加以下两行。
export GOPATH=$HOME/go
PATH=$PATH:$HOME/.local/bin:$HOME/bin:$GOPATH/bin
[root@xifeng ~]# source .bash_profile       #立即生效

三、安装git

        经测试git 1.8.3.1可以正常使用，不需要安装git 2.6.0版本。
centos7默认安装git 1.8.3.1。如果没有安装，直接yum install git即可。

[root@xifeng ~]# yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel
[root@xifeng ~]# yum install gcc perl-ExtUtils-MakeMaker       #安装依赖

四、从github下载ngrok源码

[root@xifeng ~]# cd /usr/local/                  #进入自己准备存放ngrok源码的目录。
[root@xifeng local]# git clone https://github.com/inconshreveable/ngrok.git
Cloning into ‘ngrok’…                            #克隆ngrok源码到本地ngrok目录。
remote: Counting objects: 2732, done.
remote: Total 2732 (delta 0), reused 0 (delta 0), pack-reused 2732
Receiving objects: 100% (2732/2732), 665.40 KiB | 0 bytes/s, done.
Resolving deltas: 100% (1298/1298), done.

五、为域名生成ngrok签名证书

[root@xifeng local]# cd ngrok/
[root@xifeng ngrok]# export GOPATH=/usr/local/ngrok
[root@xifeng ngrok]# openssl genrsa -out rootCA.key 2048
[root@xifeng ngrok]# openssl req -x509 -new -nodes -key rootCA.key -subj “/CN=ngrok.liuanhuaming.top” -days 5000 -out rootCA.pem
[root@xifeng ngrok]# openssl genrsa -out device.key 2048
[root@xifeng ngrok]# openssl req -new -key device.key -subj “/CN=ngrok.liuanhuaming.top” -out device.csr
[root@xifeng ngrok]# openssl x509 -req -in device.csr -CA rootCA.pem -CAkey rootCA.key -CAcreateserial -out device.crt -days 5000
#以上步骤为生成ngrok签名证书，注意将以上域名部分改为自己的。

以上签名命令完成以后，在ngrok目录下将多出6个证书文件，如下：

[root@xifeng ngrok]# ll
-rw-r–r– 1 root root 1013 Jan 4 10:13 device.crt
-rw-r–r– 1 root root 907 Jan 4 10:12 device.csr
-rw-r–r– 1 root root 1679 Jan 4 10:11 device.key
-rw-r–r– 1 root root 1675 Jan 4 10:10 rootCA.key
-rw-r–r– 1 root root 1131 Jan 4 10:11 rootCA.pem
-rw-r–r– 1 root root 17 Jan 4 10:13 rootCA.srl

将生成的证书分别替换到 assets/client/tls和assets/server/tls中，这两个目录分别存放着ngrok和ngrokd的默认证书。替换以下三个即可：

[root@xifeng ngrok]# cp rootCA.pem assets/client/tls/ngrokroot.crt
[root@xifeng ngrok]# cp device.crt assets/server/tls/snakeoil.crt
[root@xifeng ngrok]# cp device.key assets/server/tls/snakeoil.key
#以上命令为替换证书，并覆盖。

六、编译ngrokd服务端和ngrok客户端

[root@xifeng ngrok]# make release-server     #编译ngrokd，速度较慢，需等待。
  生成ngrok客户端，以下为生成windows、arm、linux版本客户端：
[root@xifeng ngrok]# GOOS=linux GOARCH=amd64 make release-client
[root@xifeng ngrok]# GOOS=windows GOARCH=amd64 make release-client
[root@xifeng ngrok]# GOOS=linux GOARCH=arm make release-client
#生成的客户端文件在ngrok/bin目录里面。
#将ngrok/bin/windows_amd64/ngrok.exe下载下来，放到windows 10系统里。

 不同平台使用不同的 GOOS 和 GOARCH，GOOS为go编译出来的操作系统 (windows,linux,darwin)，GOARCH, 对应的构架 (386,amd64,arm)：

  Linux 平台 32 位系统：GOOS=linux GOARCH=386 
  Linux 平台 64 位系统：GOOS=linux GOARCH=amd64 
Windows 平台 32 位系统：GOOS=windows GOARCH=386
Windows 平台 64 位系统：GOOS=windows GOARCH=amd64 
    MAC 平台 32 位系统：GOOS=darwin GOARCH=386 
    MAC 平台 64 位系统：GOOS=darwin GOARCH=amd64 
             ARM 平台：GOOS=linux GOARCH=arm

七、启动ngrokd服务器

[root@xifeng bin]# cd /usr/local/ngrok/bin      #进入ngrok的bin命令目录。
[root@xifeng bin]# ./ngrokd -domain=”ngrok.liuanhuaming.top” -httpAddr=”:8088″ -httpsAddr=”:8089″     
#启动ngrokd服务。-domain为ngrok服务的域名，-httpAddr为http服务端口，-httpsAddr为https服务端口。       

iptables防火墙允许8088、8089、4443端口：

-A INPUT -p tcp -m state –state NEW -m tcp –dport 8088 -j ACCEPT
-A INPUT -p tcp -m state –state NEW -m tcp –dport 4443 -j ACCEPT
#我的不使用https，所以防火墙没添加8089。4443为ngrokd服务端口。

       启动后，命令在前台运行，退出命令后服务即关闭。

后台运行ngrokd服务端命令：

nohup ./ngrokd -domain=”ngrok.liuanhuaming.top” -httpAddr=”:8088″ -httpsAddr=”:8089″ -log=”ngrok.log” &
#注意是在ngrok/bin目录下运行此命令。

      因为我命令用的./当前目录，可以使用/usr/local/ngrok/bin/ngrokd全路径，或者将命令cp至系统命令，或者加入环境变量。

命令使用的少，一般不需要添加环境变量，最好是使用全路径。

       经测试，使用nohup加&后台运行命令时，ngrokd服务不能正常使用。使用脚本可正常运行。

ngrokd服务运行脚本：
脚本放在/usr/local/ngrok/bin/目录下。

[root@xifeng bin]# vi /usr/local/ngrok/bin/ngrok.sh 
nohup ./ngrokd -domain=”ngrok.liuanhuaming.top” -httpAddr=”:8088″ -httpsAddr=”:8089″ -log=”ngrok.log” &
[root@xifeng bin]# chmod 777 ngrok.sh
#以上命令为，添加ngrok.sh脚本文件，将命令写入脚本。并赋予777权限。
[root@xifeng bin]# ./ngrok.sh &      #后台运行脚本
[1] 30181
nohup: appending output to ‘nohup.out’
[1]+ Done ./ngrok.sh

至此，ngrok服务端配置完成。

八、启动ngrok客户端

以windows 10为例：
将服务器上的/usr/local/ngrok/bin/windows_amd64/ngrok.exe下载到本地D盘根目录。
 
在本地D盘根目录创建ngrok.cfg文件，文件内容如下，域名改为自己的即可：
server_addr: “ngrok.liuanhuaming.top:4443″
trust_host_root_certs: false

启动ngrok客户端：
打开windows的cmd窗口，进入本地D盘根目录，运行以下命令：

ngrok -subdomain demo -config=./ngrok.cfg 99
#-subdomain后面的demo可以改为任意的，99为本地端口号。
因为我本地web为99端口，改为自己的即可。

Tunnel Status状态为online在线说明连接成功：
浏览器打开http://demo.ngrok.liuanhuaming.top:8088测试即可：

 

至此ngrok搭建完成！
